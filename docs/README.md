# Overview of Unity User Documentation

Welcome! This documentation walks you through the fundamental steps to install Unity, make a simple game and upload it to the internet so others can download it.

The goal of this documentation is to help beginner software developers who want to start learning game development in Unity start and finish their first game.

This documentation is divided into 4 main tasks:

| Section | Description |
|---|---|
| **1. Install Unity Editor** | Install and configure Unity Editor. |
| **2. Create a basic game scene** | Add player character, platforms, coins and a coin tracker. |
| **3.Create menus** | Add a start menu, restart screen, and quit button. |
| **4. Build and upload game** | Build the game through Unity, upload files to Itch.io for others to download. |

## How We Created Our Guide

We populated the main structure using knowledge acquired from personal experience and projects, as well as official [Unity](https://docs.unity.com/en-us "Unity documentation page") documentation.

### Using Markdown
We honed our abilities to write in Markdown over the course of writing this documentation, utilizing features like images, tables, and code blocks to increase the readability of this documentation.

### Using MkDocs
We used [MkDocs](https://www.mkdocs.org/ "MkDocs homepage") as our static site generator and [Material for MkDocs](https://squidfunk.github.io/mkdocs-material/ "Material for MkDocs repository") as our documentation theme, which simplified the process of building the site and enabled us to focus on the content of the documentation.

In addition to built-in markdown features, MkDocs allowed us to further increase the readability of this documentation with features such as admonitions.
